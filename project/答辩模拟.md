## 🛠️ 技术选型与架构（3分钟）

### 为什么选择SSM框架？

我选择Spring、Spring MVC和MyBatis这套经典组合，主要考虑：
- Spring的IoC和AOP能让代码更清晰，便于管理

### 系统架构设计

我采用了经典的三层架构：
- **表现层**：分为管理员后台和用户前台，用JSP和Layui做界面
- **业务层**：Service层处理业务逻辑，比如订单流程、权限验证
- **数据层**：用MyBatis操作MySQL数据库，设计了4个核心表

### 数据库设计思路

我设计了4个主要表：

## 💻 核心功能实现（4分钟）

### 权限控制怎么做的？

我用拦截器来控制权限：
- 管理员访问/admin路径时，检查Session里是否有管理员信息
- 普通用户访问/user路径时，验证用户登录状态
- 没有权限就跳转到登录页面

这样确保了不同角色只能访问对应的功能。

### 订单流程是怎样的？

用户预订的完整流程是：
1. 用户选择房间和日期，提交订单，状态是"待审核"
2. 管理员确认后，状态变为"已确认"，房间被预留
3. 客户到店入住，状态变为"已入住"
4. 客户退房后，状态变为"已完成"，房间释放

每个状态转换都会更新房间的可用状态，避免重复预订。

### 文件上传怎么处理的？

这是我遇到的一个技术难点。开始我把图片存在项目里，但重新部署时图片就丢了。后来我改成存储在外部目录，用Spring MVC配置静态资源映射来访问。文件名用时间戳加UUID，避免重名冲突。

### 数据统计功能

我做了入住率和收入统计：
- 入住率 = 已入住房间天数 / 总房间天数
- 可以按时间段和房间分类查看
- 用图表展示，让数据更直观

## 🙋‍♂️ 常见问题应对

### 技术问题

**Q: 为什么不用Spring Boot？**
A: 我选择SSM是为了更深入理解Spring原理。通过XML配置能更好地掌握框架机制，而且很多企业还在用SSM，学习价值高。

**Q: 如何保证数据安全？**
A: 我用了多层防护：Session认证、拦截器权限控制、MyBatis防SQL注入、前端输入验证。虽然密码还是明文，但已经考虑了主要的安全问题。

**Q: 高并发怎么处理？**
A: 目前用了Druid连接池、数据库索引优化、分页查询。如果并发更高，可以加Redis缓存、读写分离等方案。

### 业务问题

**Q: 房间冲突怎么避免？**
A: 预订时检查房间状态和时间冲突，用数据库事务保证操作原子性，订单确认后立即更新房间状态。

**Q: 入住率怎么算的？**
A: 入住率等于已入住房间天数除以总房间天数。我会统计时间段内所有房间的可用天数，然后计算实际入住的天数比例。

### 项目管理问题

**Q: 开发周期多长？**
A: 总共3个月，包括需求分析1周、系统设计1周、编码6周、测试1周、文档1周。

**Q: 遇到的最大困难？**
A: 文件上传功能。开始图片存项目里，部署时会丢失。后来改成外部存储，配置静态资源映射解决了。

## 💻 三、系统功能实现

### 3.1 管理员后台功能

**3.1.1 登录认证系统**
- 实现基于Session的登录验证
- 使用拦截器进行权限控制
- 支持记住密码功能

**核心代码展示**:
```java
@Controller
@RequestMapping("/admin")
public class AdminLoginController {
    
    @PostMapping("/doLogin")
    public String doLogin(String userName, String password, 
                         HttpSession session, Model model) {
        User admin = userService.login(userName, password, 1);
        if (admin != null) {
            session.setAttribute("adminUser", admin);
            return "redirect:/admin/index";
        }
        model.addAttribute("error", "用户名或密码错误");
        return "admin/login";
    }
}
```

### 3.3 核心技术实现

**3.3.1 权限控制机制**
```java
public class AdminLoginInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, 
                           HttpServletResponse response, 
                           Object handler) throws Exception {
        User adminUser = (User) request.getSession().getAttribute("adminUser");
        if (adminUser == null || adminUser.getUserType() != 1) {
            response.sendRedirect("/admin/login");
            return false;
        }
        return true;
    }
}
```

**3.3.2 文件上传处理**

## 🙋‍♂️ 七、答辩准备 - 常见问题与回答

### 7.1 技术相关问题

**Q1: 为什么选择SSM框架而不是Spring Boot？**

A: 选择SSM框架主要基于以下考虑：
1. **学习目的**：SSM框架能让我更深入理解Spring的核心原理，包括IoC容器、AOP、事务管理等
2. **配置理解**：通过XML配置能更好地理解框架的工作机制
3. **就业需求**：目前很多企业仍在使用SSM框架，掌握这个技术栈有助于就业
4. **循序渐进**：先掌握SSM再学习Spring Boot会更加扎实

**Q2: 如何保证系统的安全性？**

A: 系统采用了多层安全机制：
1. **认证机制**：基于Session的用户认证，防止未授权访问
2. **权限控制**：使用拦截器实现路径级别的权限验证
3. **SQL注入防护**：使用MyBatis参数化查询，避免SQL注入攻击
4. **XSS防护**：前端输入验证和后端数据过滤
5. **Session管理**：设置合理的超时时间，及时清理无效Session

**Q3: 如何处理高并发访问？**

A: 系统采用了以下优化策略：
1. **连接池管理**：使用Druid连接池，合理配置连接数量
2. **数据库优化**：添加索引、优化SQL查询语句
3. **静态资源分离**：静态文件独立存储，减轻服务器压力
4. **分页查询**：大数据量查询采用分页处理
5. **缓存机制**：后续可集成Redis缓存提高性能

**Q4: 数据库设计的考虑因素是什么？**

A: 数据库设计遵循以下原则：
1. **范式化设计**：避免数据冗余，确保数据一致性
2. **合理的外键关系**：建立表间关联，维护数据完整性
3. **索引优化**：在查询频繁的字段上建立索引
4. **字段类型选择**：根据业务需求选择合适的数据类型
5. **扩展性考虑**：预留扩展字段，支持业务发展

### 7.2 业务相关问题

**Q5: 订单状态流转的业务逻辑是什么？**

A: 订单状态流转遵循酒店业务流程：
```
待审核 → 已确认 → 已入住 → 已退房
   ↓        ↓        ↓
 已取消   已取消   (不可取消)
```
- **待审核**：用户刚下单，等待管理员确认
- **已确认**：管理员确认订单，房间预留
- **已入住**：客户办理入住，房间状态更新
- **已退房**：客户退房，订单完成，房间释放
- **已取消**：在入住前可以取消订单

**Q6: 如何计算房间入住率？**

A: 入住率计算公式：
```
入住率 = (已入住房间天数 / 总房间天数) × 100%
```
具体实现：
1. 统计时间段内所有房间的总天数
2. 统计已入住和已完成订单的入住天数
3. 计算入住率并按房间分类分组统计
4. 考虑订单时间与统计时间段的交集

**Q7: 系统如何处理房间冲突预订？**

A: 防止房间冲突的机制：
1. **实时状态检查**：预订前检查房间当前状态
2. **时间段验证**：检查预订时间是否与现有订单冲突
3. **状态同步更新**：订单确认后立即更新房间状态
4. **事务控制**：使用数据库事务确保操作原子性

### 7.3 项目管理问题

**Q8: 项目开发周期和进度安排？**

A: 项目开发分为以下阶段：
- **需求分析阶段**（1周）：调研酒店管理需求，确定功能范围
- **系统设计阶段**（1周）：数据库设计、架构设计、界面设计
- **编码实现阶段**（6周）：分模块开发，逐步实现各功能
- **测试调试阶段**（1周）：功能测试、性能测试、bug修复
- **文档编写阶段**（1周）：编写技术文档、用户手册

**Q9: 遇到的最大技术难点是什么？**

A: 最大的技术难点是文件上传功能的实现：
1. **问题**：图片上传后项目重新部署时文件丢失
2. **分析**：默认上传到项目内部目录，部署时会被覆盖
3. **解决**：配置外部存储路径，使用Spring MVC静态资源映射
4. **优化**：添加文件类型验证、大小限制、重命名机制

**Q10: 如何保证代码质量？**

A: 采用了以下代码质量保证措施：
